Destroy bank cards; it’s time for mobile currency

In the name of plastic currency we are increasing plastic cards (different types of cards like chip cards ,swipe cards and touch screen cards) , their card readers and card making machines .Millions of retailers in the world are crying like hell ,small profit margin but millions of rotation. Apart from these cards and card readers they are spending currency on cameras ,tv monitors for security purpose .

Main aim of this project is to increase plastic currency reduce plastic metals. Now it’s time for mobile currency.

 Mobile banking application  which will replace bank cards :

A  Generic mobile banking application  with Master card ,Metro card,…, Visa power and as flexible as bank cards  which will replace bank cards and their card readers. People can ask "What can I do by using this application"?

If  customer logged in as a personal account holder, by generating voucher codes customer can do deposits and withdraw cash ,cheques from atm and fund transfers to merchant account.

If customer logged in as a business account holder , by generating voucher codes customer can do deposits and withdraw cash ,cheques from atm and fund transfer to other merchant .Merchant id(tin number) is to track business transactions for tax.

People can ask "How this bank app will replace 'customer with multiple debit/credit cards' condition?".

By clicking left corner menu item a list of options will display, user needs to select virtual card option. Then user can able to see a list of virtual (debit\credit) cards which are active and tagged to that account. Then user needs to select any one card  (only one at a time) before generating voucher code.

Eg: if that customer wants to do fual purchase using debit card, and shopping  with credit card, randomly change the virtual cards check box selection before generating voucher code as shown in the below image

A scenario between  merchant and customer
How a mobile bank app customer can do payment transaction. After shopping customer needs to take merchant id (tin number) from shopkeeper fill the mid ,amount input boxes and click on generate debit voucher code.Both will get messages about transaction. Note: make sure virtual card chek box is selected before generating voucher code

Now merchant need to take voucher code from customer to validate the payment .If payment displaying with that voucher code then merchant need to take dispatch pin from customer (customer won't give  dispatch pin until stock received).

What is dispatch pin? and Why?

If you click on left corner menu button you will find a option “hot key”. Instant 3 digit key .

Once voucher code generated then merchant account will gets credited with an amount. That transaction is valid only if merchant submitted with dispatch pin.  If not  transaction is completed but customer haven’t received stock because of some delay  

 

Then customer/merchant can communicate with the bank with voucher code ,if customer/merchant  not interested with that transaction *

International centralized bank ATM and Bank Branch  (ICAB)
 Scenario between bank customer and Icab

 Customer went to unknown location to do a bank transaction with a merchant (merchant only accept cash).

Bank customer went to Icab, requested print cash but requested amount more than daily limit.

So customer communicated local branch banker through ICAB Mike.

Local banker will verify that merchant have any account with the bank to do bank transaction without printing /withdraw cash.

Or banker will communicate merchant to suggest some shares… or debentures, without printing/withdraw cash

If merchant not happy with that, banker will give additional authorization code to customer to print/withdraw cash.

Note: - ICAB aim is to provide awareness on net banking reduce paper currency
